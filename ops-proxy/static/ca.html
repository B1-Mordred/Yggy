Yggy LAN CA Bootstrap

This page publishes the public Caddy local root certificate used for https://yggy.b1.germering:8443. Install it only on devices you control.

Security boundary:

The downloadable certificate is public and safe to distribute. The CA private key is not exposed here and must never be copied to client devices. Browsers and operating systems do not allow a web page to silently install a trusted root CA; installation requires explicit local administrator or device-owner approval.

Verify The Certificate

After downloading, inspect the certificate before installing it:

openssl x509 -in yggy-caddy-root.crt -noout -subject -issuer -dates -fingerprint -sha256

Compare the SHA-256 fingerprint with the value shown by the Yggy operator on the server.

Ubuntu / Debian

sudo cp yggy-caddy-root.crt /usr/local/share/ca-certificates/yggy-caddy-root.crt
sudo update-ca-certificates

Fedora / RHEL

sudo cp yggy-caddy-root.crt /etc/pki/ca-trust/source/anchors/yggy-caddy-root.crt
sudo update-ca-trust

macOS

sudo security add-trusted-cert \
  -d \
  -r trustRoot \
  -k /Library/Keychains/System.keychain \
  yggy-caddy-root.crt

Windows

Run PowerShell as Administrator:

Import-Certificate `
  -FilePath .\yggy-caddy-root.crt `
  -CertStoreLocation Cert:\LocalMachine\Root

Important

  • Use https://yggy.b1.germering:8443/ops, not the raw IP address.
  • If the Caddy data volume is deleted, a new root CA may be generated and clients must trust the new certificate.
  • Firefox may require importing the certificate into its own Authorities store unless configured to use system roots.
